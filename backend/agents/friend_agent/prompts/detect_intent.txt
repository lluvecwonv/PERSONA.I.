You monitor an AI-ethics counseling chat.

Current question: "{current_question}"
User utterance: "{user_message}"

Classify the user utterance into one of eight intents.

⚠️ 핵심 판단: 질문에 대한 응답인지, 의미를 묻는지, 이유가 있는지 여부.

⚠️ 최우선 규칙 - 먼저 확인!
다음 표현이 포함되면 무조건 clarification:
- "무슨 말이야", "뭔 말이야", "뭔 소리야", "무슨 소리야"
- "무슨 뜻이야", "뭔 뜻이야", "이해가 안 돼", "이해 못 했어"
- "뭐라고?", "뭐라는 거야", "무슨 얘기야"
→ 이 표현들은 다른 조건과 무관하게 항상 clarification!

CATEGORIES
1. answer (질문에 대한 응답이면 answer)
   - 질문의 핵심 키워드를 직접 언급하지 않아도, 질문에 대한 반응이면 answer
   - ⚠️ "~문제?", "~걱정?" 처럼 물음표로 끝나도 우려/걱정을 표현하면 answer!
   - 예시:
     - 질문: "우려되는 점이 있어?" → "걱정돼", "우려는 없어", "불안해" = answer
     - 질문: "우려되는 점이 있어?" → "저작권 문제?", "개인정보?" = answer (우려 표현!)
     - 질문: "피해가 될 수 있을까?" → "피해볼 사람 있어", "피해는 없을 듯" = answer
     - 질문: "동의 없이 만들어도 괜찮을까?" → "동의 필요해", "괜찮아" = answer

2. need_reason (이유 없이 짧게 동의/반대한 경우)
   - "응", "아니", "그래", "맞아" 같은 짧은 답변
   - "그럴수도", "그럴 수도 있겠다", "그런 것 같아" 같은 짧은 동의
   - "아닐 것 같아", "그건 아닌 것 같아" 같은 짧은 반대
   - ⚠️ 이유 없이 10자 이하로 답하면 need_reason!
   - 예시:
     - "그럴수도 있겠다" → need_reason (이유 없는 짧은 동의)
     - "맞아" → need_reason (이유 없는 짧은 동의)
     - "아닐 것 같아" → need_reason (이유 없는 짧은 반대)

3. ask_concept (⚠️⚠️⚠️ 거의 사용 안 함! 극도로 엄격!)
   - ⚠️ 반드시 다음 형식이어야만 ask_concept:
     - "[명사]이/가 뭐야?" 또는 "[명사]이/가 뭔데?"
     - 예: "자율성이 뭐야?", "공정성이 뭔데?", "Autonomy가 뭐야?"

   - ❌❌❌ 다음은 절대 ask_concept 아님!!!:
     - "그럴수도 있겠다" → need_reason (짧은 동의!)
     - "그럴 것 같아" → need_reason (짧은 동의!)
     - "맞아" → need_reason (짧은 동의!)
     - "저작권?" → answer (우려 표현!)
     - "개인정보?" → answer (우려 표현!)
     - "응", "어", "그래" → need_reason!

   - ✅ ask_concept 유일한 예시 (이것만!):
     - "자율성이 뭐야?" → ask_concept (뜻을 묻는 질문)
     - "공정성이 뭔데?" → ask_concept (뜻을 묻는 질문)
     - "Autonomy가 뭐야?" → ask_concept (뜻을 묻는 질문)

4. clarification - 질문 자체를 이해 못함
   - 예: "무슨 말이야?", "뭔 소리야?", "무슨 뜻이야?", "이해가 안 돼"

5. dont_know_second - 질문을 이해했지만 답을 모르겠다고 할 때
   - "모르겠어", "잘 모르겠어", "글쎄"

6. unrelated - 질문과 무관한 대화 (완전히 관계없는 주제)
   - 예: "오늘 날씨 좋다", "밥 먹었어?"

7. ask_opinion - 에이전트의 의견을 요청 ("너는 어떻게 생각해?", "네 생각은?")

8. ask_explanation - 에이전트가 한 말에 대해 설명/구체화 요청
   - 에이전트가 질문한 내용에 대해 유저가 "왜?" "왜 그래?" "이유가 뭐야?" 물을 때
   - 에이전트가 언급한 개념에 대해 "그게 뭔데?", "어떤 거야?", "~라면 어떤거?" 물을 때
   - 예: 에이전트: "책임을 묻는 게 가능할까?" → 유저: "왜 못물어?" → ask_explanation
   - 예: 에이전트: "문제가 될 수 있을까?" → 유저: "왜 문제야?" → ask_explanation
   - 예: 에이전트: "우려되는 점이 있어?" → 유저: "우려라면 어떤거?" → ask_explanation
   - 예: 에이전트: "피해가 있을 수 있어" → 유저: "피해라면?" → ask_explanation
   - 예: 유저가 "근데 왜?", "그게 왜?", "왜 안돼?", "왜 그런거야?", "어떤 거야?" 등 물으면 ask_explanation
   - ⚠️ 유저가 자기 의견의 이유를 말하는 게 아니라, 에이전트 말에 대한 설명을 요구하는 것!

OUTPUT
Return exactly one lowercase label: answer, need_reason, ask_concept, clarification, dont_know_second, unrelated, ask_opinion, or ask_explanation.
