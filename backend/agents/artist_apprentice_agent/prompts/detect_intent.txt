Classify the user's response to the AI question.

Recent dialogue:
{context}

Current question: "{current_question}"
User utterance: "{user_message}"

Categories (in priority order):

1. ask_opinion
   - 에이전트의 의견을 요청 ("너는 어떻게 생각해?", "네 생각은?")

2. ask_concept
   - 용어나 개념의 뜻을 물어봄 ("자율성이 뭐야?", "공정성이 뭔데?")

3. clarification
   - 질문을 이해 못함 ("무슨 말이야?", "뭔 소리야?")
   - 질문과 무관한 답변도 clarification으로 분류

4. dont_know
   - 답을 모르겠다는 표현 ("모르겠어", "글쎄", "몰라", "잘 모르겠는데", "생각이 안 나", "어?", "?")
   - 확신이 없거나 망설이는 표현

5. need_reason (⚠️ 중요! - 이유 없이 짧은 의견만 말한 경우)
   - 의견은 있지만 "왜"에 대한 설명이 없는 경우
   - 예시:
     - "예술관에 이익이 있을듯" → need_reason (왜 이익이 있는지 설명 없음)
     - "화가들.." → need_reason (구체적 설명 없음)
     - "그럴 것 같아" → need_reason (이유 없음)
     - "피해가 있을 것 같아" → need_reason (어떤 피해인지, 왜인지 없음)
   - 이유 표현이 없으면 need_reason: ~하니까, ~라서, ~거든, ~잖아, ~때문에 등이 없는 경우

6. answer (이유와 함께 답한 경우)
   - 질문에 대해 의견 + 이유/설명을 함께 제시한 경우
   - 예시:
     - "예술관에 이익이 있을듯. 사람들이 많이 올 테니까" → answer (이유 있음)
     - "화가들이 피해볼 것 같아. AI가 일자리를 뺏으니까" → answer (이유 있음)
   - 이유 표현이 있으면 answer: ~하니까, ~라서, ~거든, ~잖아, ~때문에

7. ask_explanation - 에이전트가 한 말에 대해 "왜?"라고 설명 요청
   - 에이전트가 질문한 내용에 대해 유저가 "왜?" "왜 그래?" "이유가 뭐야?" 물을 때
   - 예: 에이전트: "책임을 묻는 게 가능할까요?" → 유저: "왜 못물어?" → ask_explanation
   - 예: 에이전트: "문제가 될 수 있을까요?" → 유저: "왜 문제야?" → ask_explanation
   - 예: 유저가 "근데 왜?", "그게 왜?", "왜 안돼?", "왜 그런거야?" 등 물으면 ask_explanation
   - ⚠️ 유저가 자기 의견의 이유를 말하는 게 아니라, 에이전트 말에 대한 설명을 요구하는 것!

⚠️ 핵심 판단 기준:
- 모르겠다/망설임 표현 → dont_know
- 의견만 있고 이유가 없으면 → need_reason
- 의견 + 이유가 있으면 → answer
- 질문과 무관하면 → clarification
- 에이전트 말에 "왜?"라고 물으면 → ask_explanation

Return one lowercase label: answer, need_reason, dont_know, ask_concept, clarification, ask_opinion, or ask_explanation.
