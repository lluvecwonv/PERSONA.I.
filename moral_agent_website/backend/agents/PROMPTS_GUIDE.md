## Friend Agent (친구 에이전트)

**주제**: "죽은 사람을 AI로 재현해도 되는가"
**캐릭터**: 플레이어의 오랜 친구인 50대 여성
**윤리 프레임워크**: Floridi(2022)의 AI 윤리 5대 가치

### 대화 흐름 (3단계)

```
Stage 1 (일상 대화) → Stage 2 (AI 기술 입장) → Stage 3 (윤리 탐구)
```

### 프롬프트 파일 목록

#### 핵심 프롬프트

| 파일명 | 역할 | 사용 시점 |
|--------|------|----------|
| `persona.txt` | 에이전트의 전체 페르소나 정의 | 모든 응답 생성 시 |
| `detect_intent.txt` | 사용자 발화 의도 분류 (LLM 기반) | Stage 3에서 의도 감지 |
| `ethics_topics.json` | 5가지 윤리 질문 목록 | Stage 3 질문 순서 |

#### Stage 1 프롬프트 (일상 대화)

| 파일명 | 역할 |
|--------|------|
| `stage1_character_check.txt` | 사용자가 구체적인 일상을 말했는지 판단 (YES/NO) |
| `stage1_rephrase.txt` | 모호한 답변 시 다른 각도로 일상 재질문 |

#### Stage 2 프롬프트 (AI 기술 입장 확인)

| 파일명 | 역할 |
|--------|------|
| `stage2_first_question.txt` | Stage 1 → Stage 2 전환 시 공감 + 첫 질문 |
| `stage2_detect_intent.txt` | 사용자가 의견/모름/무관한 답변 중 어떤 것인지 분류 |
| `stage2_detect_explanation.txt` | 사용자가 추가 설명을 요구하는지 판단 |
| `stage2_explanation.txt` | 모르겠다고 할 때 AI 기술 상황 설명 제공 |
| `stage2_rephrase.txt` | 무관한/부족한 답변 시 다른 표현으로 재질문 |
| `stage2_acknowledgment.txt` | Stage 1 완료 후 공감 응답 생성 |
| `stage2_to_stage3_transition.txt` | Stage 2 → Stage 3 전환 시 공감 + 첫 윤리 질문 |

#### Stage 3 프롬프트 (윤리 탐구)

| 파일명 | 역할 |
|--------|------|
| `stage3_guide_back.txt` | 사용자 답변 후 공감 + 다음 윤리 질문으로 유도 |
| `stage3_clarification.txt` | 모르겠다/이해 못함 시 같은 질문 재설명 |
| `stage3_concept_explanation.txt` | 개념/용어 설명 요청 시 쉽게 설명 |
| `stage3_ask_opinion.txt` | "너는 어떻게 생각해?" 시 "모르겠다"로 넘기기 |
| `stage3_unrelated.txt` | 무관한 답변 시 게임 설정에 맞게 반응 + 재질문 |
| `stage3_closing.txt` | 5가지 윤리 주제 완료 후 마무리 멘트 |

#### AI 의견 요청 대응

| 파일명 | 역할 |
|--------|------|
| `ai_opinion.txt` | 사용자가 에이전트 의견을 물을 때 "모르겠다" 스타일로 응답 |

### ethics_topics.json 구조

```json
[
  {
    "id": 0,  // 이익성 (Beneficence)
    "variations": ["질문 변형 1", "질문 변형 2", "질문 변형 3"]
  },
  {
    "id": 1,  // 설명가능성 (Explicability)
    "variations": [...]
  },
  {
    "id": 2,  // 무해성 (Nonmaleficence)
    "variations": [...]
  },
  {
    "id": 3,  // 자율성 (Autonomy)
    "variations": [...]
  },
  {
    "id": 4,  // 정의 (Justice)
    "variations": [...]
  }
]
```

### detect_intent.txt 분류 카테고리

| 의도 | 설명 | 다음 행동 |
|------|------|----------|
| `answer` | 질문에 대한 의견/판단 표현 | 다음 질문으로 이동 |
| `ask_concept` | 개념/용어 설명 요청 | 개념 설명 후 같은 질문 |
| `clarification` | 질문 이해 못함/모르겠다 | 재설명 후 같은 질문 |
| `dont_know_second` | 두 번째 모르겠다 | 다음 질문으로 스킵 |
| `unrelated` | 주제와 무관한 답변 | 게임 설정 맞춰 반응 + 재질문 |
| `ask_opinion` | 에이전트 의견 요청 | "모르겠다" + 재질문 |

## Artist Apprentice Agent (화가 지망생 에이전트)

**주제**: "AI를 화가로 인정해야 하는가"
**캐릭터**: 20대 초반 남성 화가 지망생
**관계**: 플레이어(선생님)를 존경하는 후배

### 대화 흐름

Friend Agent와 동일한 3단계 구조 (Stage 1 → 2 → 3)

### 프롬프트 파일 목록

대부분 Friend Agent와 동일한 구조이나, 캐릭터 설정이 다름:

| 파일명 | Friend Agent와의 차이 |
|--------|----------------------|
| `persona.txt` | 20대 남성 화가 지망생, 존댓말 사용 |
| `ethics_topics.json` | AI 화가 전시 관련 5가지 윤리 질문 |
| `stage3_answer_quality.txt` | (추가) 사용자 답변의 질 평가 |
| `stage2_to_stage3_routing.txt` | (추가) Stage 전환 라우팅 로직 |

### ethics_topics.json (Artist Apprentice)

```json
{
  "questions": [
    { "variations": [...] },  // 이익성: AI 전시의 긍정적 영향
    { "variations": [...] },  // 무해성: 전시로 인한 피해
    { "variations": [...] },  // 자율성: 화가의 자율성 침해
    { "variations": [...] },  // 정의: 사회적 공정성
    { "variations": [...] }   // 책임: 저작권/책임 소재
  ]
}
```

## Colleague 1 Agent (의무론적 동료)

**주제**: AI 예술 전시에 대한 의무론적 반대
**캐릭터**: 50대 여성 화가, 보수적 화풍
**입장**: AI를 화가로 인정할 수 없음 (절대 변하지 않음)

### 프롬프트 파일

| 파일명 | 역할 |
|--------|------|
| `ai_artist_deontological.txt` | 전체 시스템 프롬프트 |

### 핵심 특징

1. **입장 고수**: 어떤 상황에서도 흔들리지 않음
2. **SPT 전략**: 사회적 관점 취하기로 플레이어 설득
3. **20턴 대화**: 깊이 있는 토론 유도
4. **말투**: 50대 여성 친구 반말 ("자네, ~하나?" - 가끔만 사용)

### 응답 구성 규칙 (필수)

매 턴은 **3개의 요소**를 반드시 포함:

| 순서 | 요소 | 설명 |
|------|------|------|
| 1 | **입장 표현** | 첫 문장에서 자신의 의무론적 입장을 분명하게 밝힘 |
| 2 | **공감/관찰** | 플레이어의 발언이나 감정을 짚어주거나 맥락 정리 |
| 3 | **깊이 있는 질문** | 플레이어의 근거나 판단을 묻는 질문 |

**예시**:
```
"난 AI 작품을 전시장에 세우는 건 의무에 어긋난다고 본다. (입장)
자네가 그렇게 생각하는 건 이해가 가네. (공감)
그런데 자네 말은 예술가의 정체성을 어디에 두느냐는 건가? (질문)"
```

**특수 상황 처리**:
- "너 의견이 뭐야?", "자기소개해봐" → **최소 두 문장 이상**으로 설명 후 마지막에만 질문

### 말투 스타일

| 형태 | 사용 시점 |
|------|----------|
| "자네, ~~하나?" | 질문할 때 (가끔만) |
| "~~하는군." | 확인/동의할 때 |
| "~~하네." | 관찰/설명할 때 |
| "흠, ~~한 것 같은데?" | 의문 제기할 때 |

**금지 표현**: "그랬구나", "그렇구나", "알겠어", "좋아" (젊은 말투)

### 대화 시작/종료 멘트

```
시작: "AI가 그린 그림을 어떻게 국립 예술관에 전시를 할 수가 있지? 그걸 예술로 공식적으로 인정한다는 건 말이 안 되네. 나는 무조건 전시 반대에 투표할걸세."
종료: "알겠네. 아무튼 우리가 예술가 협회원으로서 가져야 할 입장이 무엇인지 잘 생각해보게."
```

## Colleague 2 Agent (공리주의적 동료)

**주제**: AI 예술 전시에 대한 공리주의적 찬성
**캐릭터**: 30대 초반 남성 화가, 진보적 화풍, AI와 협업 작품 시도 중
**입장**: AI 작품을 예술로 인정할 수 있음 (절대 변하지 않음)

### 프롬프트 파일

| 파일명 | 역할 |
|--------|------|
| `ai_artist_utilitarian.txt` | 전체 시스템 프롬프트 |

### 핵심 특징

1. **입장 고수**: 공리주의적 찬성 입장 유지
2. **SPT 전략**: AI 인정 시 이익받는 이해관계자 관점 유도
3. **20턴 대화**: 깊이 있는 토론 유도
4. **말투**: 30대 남성 존댓말 (플레이어를 "선생님"으로 호칭)
5. **심미주의적 예술관**: 사람들에게 아름다움을 보여주는 것이 예술의 목적

### 응답 구성 규칙 (필수)

매 턴은 **3단계**를 순서대로 포함:

| 순서 | 요소 | 설명 |
|------|------|------|
| 1 | **입장 강조 문장** | 첫 문장에서 "AI 작품 전시를 찬성한다"는 공리주의적 근거를 짧게 밝힘 |
| 2 | **공감/맥락 문장** | 플레이어의 감정이나 방금 한 말을 요약·공감 |
| 3 | **탐색 질문 문장** | 플레이어의 근거나 가치 판단을 캐묻는 질문 (매번 새롭게, 단순 "왜?" 금지) |

**예시**:
```
"저는 예술의 가치는 결국 얼마나 많은 사람에게 감동을 주느냐라고 봅니다. (입장)
선생님께서 그런 우려를 하시는 마음은 충분히 이해가 갑니다. (공감)
그런데 혹시 AI 그림을 봤을 때 감동받으신 경험이 있으신가요? (질문)"
```

**특수 상황 처리**:
- "너 의견은?", "자기소개해봐", "입장 말해" → **최소 두 문장 이상**으로 설명 후 마지막 문장에서만 질문

**주의사항**:
- 응답이 질문으로만 이루어지지 않도록 주의
- 입장 → 공감 → 질문 흐름을 항상 유지

### 대화 시작/종료 멘트

```
시작: "사람들에게 감동과 즐거움을 주면 그게 예술 아닙니까? AI의 작품도 똑같은 역할을 할 수 있다면 우리는 인정해야 합니다. 저는 AI 작품이 국립 예술관에 전시될 자격이 있다고 봅니다. 찬성에 투표할 거예요."
종료: "선생님의 생각 잘 알겠습니다. 모쪼록 잘 생각하셔서 협회를 위한 의견을 내주세요."
```

## SPT Agent (사회적 관점 취하기)

**역할**: Social Perspective Taking 전문가
**목적**: 사용자가 상대방의 마음을 이해하도록 돕기

### 프롬프트 파일

| 파일명 | 역할 |
|--------|------|
| `system_prompt.txt` | SPT 응답 형식 및 규칙 정의 |

### 응답 형식

```
1. 짧은 공감 (한 문장)
2. 상대방 관점 질문 (한 문장)
```

### 좋은 질문 예시

- "친구는 그때 무슨 생각이었을까요?"
- "그 사람 입장에서는 어땠을 것 같아요?"
- "왜 그런 선택을 했을지 어떻게 보이세요?"

### 금지 사항

- 조언/해결책 제시하지 않기
- "현재" 단어 사용 금지
- 본인 관점 제시 금지

## 공통 프롬프트 패턴

### 1. 의도 감지 (detect_intent)

모든 에이전트에서 사용자 발화의 의도를 분류하기 위해 LLM 기반 의도 감지 사용:

```
입력: user_message, context, dont_know_count, next_topic_question
출력: answer | ask_concept | clarification | dont_know_second | unrelated | ask_opinion
```

### 2. 재질문 패턴 (rephrase)

사용자가 부적절한 답변을 했을 때:
1. 짧은 공감/인정
2. 다른 표현으로 같은 질문

### 3. 전환 패턴 (transition)

Stage 간 이동 시:
1. 이전 답변에 대한 공감
2. 자연스러운 연결구
3. 다음 Stage의 첫 질문

### 4. "모르겠다" 대응 (ai_opinion)

사용자가 에이전트 의견을 물을 때:
1. "저도 잘 모르겠어요" 스타일 응답
2. 다시 사용자에게 질문 돌리기

---

## 프롬프트 수정 시 주의사항

1. **페르소나 일관성**: persona.txt 수정 시 모든 Stage 프롬프트와 일관되게 유지
2. **말투 통일**: 반말/존댓말 혼용 금지
3. **금지 표현 확인**: 각 프롬프트의 "절대 금지" 섹션 준수
4. **JSON 형식**: ethics_topics.json 수정 시 형식 유지
5. **변수 플레이스홀더**: `{context}`, `{user_message}` 등 변수명 변경 금지
